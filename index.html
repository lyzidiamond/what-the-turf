What does it mean, spatial data analysis?
This is a classic example of spatial analysis.
Physician John Snow created this map during London's cholera outbreak of 1854.
Snow plotted cholera cases in the Soho area of London around Broad Street and noticed a cluster around the water pump.
What did this mean? Through this map, he discovered that cholera was a water-borne disease...
... disproving the commonly-held beliefs in miasma at the time.

But it has historically been very difficult to conduct these types of analyses programmatically in the browser.
Why? Think about the forces at play here.
It's the same reason websites will load slowly when trying to display a large image.
Browsers do not have as much power as a machine's processor.
Plus, think about geographic data. What works well for desktop GIS doesn't work as wel for browsers and vice-versa.
The types of data formats we use for desktop GIS are optimized for processing speed with the assumption that there will be lots of data.
The types of data formats we use for displaying on web maps are optimized for speed of display with the assumption that there will be not as much data.
But I digress.
It has historically been very difficult to conduct spatial analysis programmatically in the browser.
Turf changes all that.
Going back for a second, Turf is GIS for web maps.
What is the language of web maps?
JavaScript!
But Turf does not display web maps for us.
It does not add tiles or enable zooming and panning.
What it does do is:
1. Take GeoJSON as input
2. Conduct analysis on that GeoJSON
3. Return GeoJSON as output
And it does it with JavaScript.

Just a brief introduction to programming concepts...
Variables: placeholders.
var x = 4
Objects: buckets of different values that can be grouped together.
Let's say we have an object dog. What does a dog have?
A dog has a name.
A dog has an age.
A dog has a breed.
So we can use our dog object to store all of this information together.
var dog = {
  name: 'Wiggles',
  age: 2,
  breed: 'corgi'
}
Now, when we want to find the dog's age, we can access them this way:
dog.age; // 2
Classes: a way to create similar objects over and over again.
Let's say we want to have five dogs. Each dog has a name, age, and breed.
Instead of re-defining what a dog is each time, we can make a dog class.
function Dog(name, age, breed) {
  this.name = name;
  this.age = age;
  this.breed = breed;
}
Now when we create a new dog:
var newDog = new Dog('Spinner', 4, 'german shepherd');
newDog.name; // 'Spinner'
Cool. But what are all these parentheses about?
In a very stripped-down way, JavaScript works by taking values, doing something to them, and returning the result.
Does anyone remember function boxes from elementary school?
This is the same thing you're doing in JavaScript.
When we made 
newDog
, we passed three values into our 
Dog
function box...
... and it returned a 
newDog
!
